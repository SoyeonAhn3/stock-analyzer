1. 전체 흐름 요약
================================================================================

  사용자: 브라우저에서 "NVDA" 검색
          |
          v
    [React 프론트엔드] (localhost:3000)
          |  HTTP 요청
          v
    [FastAPI 백엔드] (localhost:8000)
          |
          v
    [STEP 1] Quick Look (즉시, AI 없음, 1~2초)
             "숫자를 모아오는 단계"
          |
          v
    [STEP 2] Deep Analysis (1~2분, AI 사용)
             "숫자를 해석하는 단계"
          |
          v
    [STEP 3] JSON 응답 → React가 화면에 표시

2. 아키텍처 개요 (React + FastAPI)
================================================================================

  ┌─────────────┐     HTTP/JSON      ┌──────────────┐
  │   React     │ ─────────────────→ │   FastAPI    │
  │  (브라우저)  │ ←───────────────── │   (서버)     │
  │             │                    │              │
  │  화면 담당   │                    │  데이터+AI   │
  │  TypeScript │                    │  Python      │
  └─────────────┘                    └──────┬───────┘
                                            │
                                    ┌───────┴───────┐
                                    │               │
                              data/ 모듈      agents/ 모듈
                              (Phase 1~2)     (Phase 3~5)
                                    │               │
                              api_client.py   claude_client.py
                                    │               │
                              외부 금융 API     Claude API

  [핵심 포인트]
  - React는 FastAPI의 엔드포인트만 호출 (Python 함수 직접 호출 불가)
  - FastAPI는 기존 data/agents 함수를 그대로 호출 (코드 변경 없음)
  - React ↔ FastAPI 사이의 데이터는 전부 JSON


================================================================================
3. STEP 1 — Quick Look (데이터 수집)
================================================================================

[목적]
  시세, 차트, 재무지표, 기술지표를 외부 API에서 수집한다.
  AI를 사용하지 않으므로 비용 없이 1~2초 만에 완료된다.

[호출 흐름]

  React (브라우저)
    |
    |-- fetch("/api/quote/NVDA")         → FastAPI → quote.py
    |-- fetch("/api/history/NVDA")       → FastAPI → history.py
    |-- fetch("/api/fundamentals/NVDA")  → FastAPI → fundamentals.py
    |-- fetch("/api/technicals/NVDA")    → FastAPI → technicals.py
    |
    |  (4개 요청을 Promise.all로 병렬 실행)
    |
    v
  FastAPI (backend/routers/quote.py)
    |
    |-- get_quote("NVDA") ----------+
    |-- get_history("NVDA") ---------+--- api_client.py --- 외부 API
    |-- get_fundamentals("NVDA") ----+       |
    |-- get_technicals("NVDA") ------+       |-- Finnhub    (시세, 뉴스)
                                             |-- yfinance   (재무, 차트)
                                             |-- TwelveData (기술지표)
                                             |-- FMP        (섹터, 재무제표)
                                             |-- Finviz     (섹터 PE)
                                             |-- FRED       (금리, CPI)

[api_client.py의 역할]
  데이터 유형별로 우선순위에 따라 API를 호출한다.
  1순위 실패 시 자동으로 2순위 폴백, 그래도 실패 시 3순위 시도.
  전부 실패하면 해당 항목 "데이터 없음" 반환.

  데이터 유형      1순위        2순위         3순위
  -------------------------------------------------------
  시세             Finnhub      yfinance      -
  주가 히스토리    yfinance     TwelveData    -
  기업 재무        yfinance     FMP           -
  기술적 지표      TwelveData   Python 계산   -
  뉴스             Finnhub      -             -

[API 응답 예시: GET /api/quote/NVDA]

      "ticker": "NVDA",
      "price": 142.50,
      "change": 3.21,
      "change_percent": 2.3,
      "volume": 342100000,
      "market_cap": 3520000000000,
      "source": "finnhub"
  }

[API 응답 예시: GET /api/technicals/NVDA]

  {
      "ticker": "NVDA",
      "rsi": {"value": 62.4, "signal": "neutral"},
      "macd": {"histogram": 1.25, "signal": "bullish", "detail": "..."},
      "bollinger": {
          "upper": 148.12, "middle": 141.05, "lower": 133.98,
          "position": "upper_half", "signal": "neutral"
      },
      "ma50": {"value": 135.20, "vs_price": "+5.4%", "signal": "bullish"},
      "ma200": {"value": 118.50, "vs_price": "+20.3%", "signal": "bullish"},

4. STEP 2 — Deep Analysis (AI 분석)
================================================================================

[목적]
  Quick Look에서 수집한 숫자를 AI(Claude)가 해석하여
  최종 BUY/HOLD/SELL 판단을 내린다.

[사용자 트리거]
  Quick Look 화면에서 "AI 분석 실행" 버튼 클릭

[호출 흐름]

  React (브라우저)
    |
    |-- fetch("/api/analysis/NVDA", {method: "POST"})
    |   (1~2분 대기, 로딩 UI 표시)
    |
    v
  FastAPI (backend/routers/analysis.py)
    |
    |-- 먼저 quick_look_data 수집 (get_quote + get_fundamentals + get_technicals)
    |
    v
  orchestrator.py (지휘자)
    |
    |-- [1단계: 병렬 실행] --------------------------------+
    |     |                                                |
    |     |-- news_agent.py                                |
    |     |     1. Finnhub에서 뉴스/애널리스트 수집          |
    |     |     2. Claude에게 "뉴스 감성 분석해줘"           |
    |     |     => Claude 1회 호출                          |
    |     |                                                |
    |     |-- data_agent.py                                |
    |     |     1. quick_look_data 재사용 (API 호출 없음!)   |
    |     |     2. 섹터 평균 PE만 추가 수집 (Finviz)          |
    |     |     3. Claude에게 "이 숫자들 해석해줘"            |
    |     |     => Claude 1회 호출                          |
    |     |                                                |
    |     |-- macro_agent.py                               |
    |     |     1. FRED API에서 금리/CPI/실업률 수집          |
    |     |     2. Claude에게 "경제 상황 분석해줘"            |
    |     |     => Claude 1회 호출                          |
    |     |                                                |
    |     +------------------------------------------------+
    |     3개가 동시에 실행됨 (asyncio.gather)
    |     순차: 30초 x 3 = 90초 vs 병렬: 30초 = 3배 빠름
    |
    |-- [2단계: Graceful Degradation 판정] ----------------+
    |     |                                                |
    |     |  성공 3/3 => 정상 진행                           |
    |     |  성공 2/3 => 진행, 신뢰도 1단계 하향              |
    |     |  성공 1/3 => 진행, 신뢰도 low 고정               |
    |     |  성공 0/3 => 분석 중단, 에러 반환                 |
    |     +------------------------------------------------+
    |
    |-- [3단계: 교차검증]
    |     |-- cross_validation.py
    |     |     Claude에게 "모순 있는지 찾아줘"
    |     |     => Claude 1회 호출
    |
    |-- [4단계: 최종 판단]
    |     |-- analyst_agent.py
    |     |     Claude에게 "최종 판단 내려줘"
    |     |     => Claude 1회 호출
    |
    v
  JSON 응답 → React

[API 응답 예시: POST /api/analysis/NVDA]

  {
      "ticker": "NVDA",
      "agent_results": {
          "news":  {"sentiment": "positive", "earnings": "beat", ...},
          "data":  {"valuation": "fair", "rsi": "neutral-bull", ...},
          "macro": {"rate_outlook": "hold", "sector_trend": "growth", ...}
      },
      "agent_status": {
          "news": "success",
          "data": "success",
          "macro": "success"
      },
      "cross_validation": {
          "conflicts": ["기술지표 과매수 vs 실적 호조"],
          "agreements": ["성장 모멘텀 3개 Agent 모두 동의"]
      },
      "analyst": {
          "verdict": "BUY",
          "confidence": "high",
          "bull_case": "AI 수요 폭발 + 실적 서프라이즈",
          "bear_case": "밸류에이션 부담 + 금리 리스크",
          "summary": "성장 모멘텀이 밸류에이션 부담을 상회",
          "disclaimer": "본 분석은 AI가 생성한 참고 자료이며..."
      },
      "errors": []
  }

[모든 Claude 호출 경로]

  news_agent.py ------+
  data_agent.py -------+-- claude_client.py ---> Claude API
  macro_agent.py ------+       |
  cross_validation.py -+       |-- 호출마다 usage_tracker.increment()
  analyst_agent.py ----+       |-- JSON 파싱 실패 시 raw_output 저장
                               |-- 하루 100회 하드 리밋 체크

6. 파일 간 역할 분담 정리
================================================================================

[파일]                    [역할]                         [비유]
--------------------------------------------------------------------------
--- React (프론트엔드) ---
App.tsx                   페이지 라우팅                   교통 경찰
pages/QuickLook.tsx       Quick Look 화면                 접수 창구
pages/AIAnalysis.tsx      AI 분석 결과 화면               분석 보고서
hooks/useQuote.ts         API 호출 + 로딩/에러 관리       전화 주문
theme/ThemeProvider.tsx   Dark/Light 테마 관리            조명 담당

--- FastAPI (백엔드) ---
backend/main.py           서버 시작 + CORS                정문 경비
backend/routers/*.py       HTTP 요청 → Python 함수 연결   카운터 직원

--- Python 데이터 계층 ---
orchestrator.py           분석 흐름 제어 + 실패 처리      팀장
news_agent.py             뉴스 감성 분석                  뉴스 담당자
data_agent.py             재무/기술지표 해석              재무 담당자
macro_agent.py            거시경제 분석                   경제 담당자
cross_validation.py       모순 탐지                      감사관
analyst_agent.py          최종 BUY/HOLD/SELL 판단         수석 애널리스트
claude_client.py          Claude API 호출 공통 함수       전화기
api_client.py             외부 데이터 API 통합 + 폴백     데이터 수집반
usage_tracker.py          일일 AI 사용량 추적             계량기

[핵심 설계 원칙]
- React는 API 엔드포인트만 알면 됨 (Python 함수 존재를 모름)
- FastAPI는 기존 함수를 감싸기만 함 (로직 변경 없음)
- data/agents 모듈은 프론트엔드 존재를 모름 (순수 Python)
- 각 Agent는 자기 분야만 담당 (다른 Agent 존재를 모름)
- 모든 Claude 호출은 claude_client.py 한 곳을 통과
- quick_look_data는 한 번만 수집되고 모든 곳에서 재사용

8. 비용 요약
================================================================================

[1회 Deep Analysis]
  Claude 호출: 5회
  예상 토큰: Input ~11,700 + Output ~3,500
  예상 비용: ~$0.09 (약 130원)

[일일 사용 시나리오]
  가볍게 (5회/일)  => 월 $9.9   (약 14,000원)
  보통   (10회/일) => 월 $19.8  (약 28,000원)
  적극적 (20회/일) => 월 $39.6  (약 56,000원)
  하드 리밋 (100회/일) => 월 $198 (약 280,000원)

[비용 절약 장치]
  - 일일 100회 하드 리밋
  - Data Agent는 quick_look_data 재사용 (API 재호출 없음)
  - 캐싱 (같은 종목 1시간 이내 재분석 시 캐시 사용)
  - Quick Look은 AI 미사용 (무료)
  - 80회 도달 시 경고 표시

[인프라 비용]
  - FastAPI 서버: 로컬 실행 시 무료
  - React: 로컬 실행 시 무료
  - 외부 API: 전부 무료 티어 사용
