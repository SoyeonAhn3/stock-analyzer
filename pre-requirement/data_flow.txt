1. 전체 흐름 요약
================================================================================

  사용자: "NVDA" 입력
          |
          v
    [STEP 1] Quick Look (즉시, AI 없음, 1~2초)
             "숫자를 모아오는 단계"
          |
          v

             "숫자를 해석하는 단계"
          |
          v
    [STEP 3] 화면에 결과 표시


================================================================================
2. STEP 1 — Quick Look (데이터 수집)
================================================================================

[목적]
  시세, 차트, 재무지표, 기술지표를 외부 API에서 수집한다.
  AI를 사용하지 않으므로 비용 없이 1~2초 만에 완료된다.

[호출 흐름]

  app.py (Streamlit 메인)
    |
    |-- quote.py ----------+
    |-- history.py ---------+--- api_client.py --- 외부 API
    |-- fundamentals.py ----+       |
    |-- technicals.py ------+       |-- Finnhub    (시세, 뉴스)
                                    |-- yfinance   (재무, 차트)
                                    |-- TwelveData (기술지표)
                                    |-- FMP        (섹터, 재무제표)
                                    |-- FRED       (금리, CPI)

[api_client.py의 역할]
  데이터 유형별로 우선순위에 따라 API를 호출한다.
  1순위 실패 시 자동으로 2순위 폴백, 그래도 실패 시 3순위 시도.
  전부 실패하면 해당 항목 "데이터 없음" 반환.

  데이터 유형      1순위        2순위         3순위
  -------------------------------------------------------
  시세             Finnhub      yfinance      -
  주가 히스토리    yfinance     TwelveData    -
  기업 재무        yfinance     FMP           -
  기술적 지표      TwelveData   Python 계산   -
  뉴스             Finnhub      -             -

[결과물: quick_look_data (dict)]

  quick_look_data = {
      "ticker": "NVDA",

      # quote.py
      "price": 950.02,
      "change": +15.30,
      "change_pct": +1.64,

      # history.py
      "history": [{"date": "2026-04-09", "close": 950.02}, ...],

      # fundamentals.py
      "pe": 35.2,
      "eps": 4.05,
      "market_cap": 2340000000000,
      "sector": "Technology",
      "de_ratio": 0.41,

      # technicals.py
      "rsi": 72.5,
      "macd": {"value": 12.3, "signal": 10.1},
      "ma50": 890.0,
      "ma200": 750.0,
  }

  이 데이터는 화면에 바로 표시되고,
  Deep Analysis에서도 그대로 재사용된다 (API 재호출 없음).


================================================================================
3. STEP 2 — Deep Analysis (AI 분석)
================================================================================

[목적]
  Quick Look에서 수집한 숫자를 AI(Claude)가 해석하여
  최종 BUY/HOLD/SELL 판단을 내린다.

[사용자 트리거]
  Quick Look 화면에서 "AI 분석 시작" 버튼 클릭

[호출 흐름]

  app.py
    |
    v
  orchestrator.py (지휘자)
    |
    |-- [1단계: 병렬 실행] --------------------------------+
    |     |                                                |
    |     |-- news_agent.py                                |
    |     |     1. Finnhub에서 뉴스/애널리스트 수집          |
    |     |     2. Claude에게 "뉴스 감성 분석해줘"           |
    |     |     3. 결과: 감성, 실적, 애널리스트 종합          |
    |     |     => Claude 1회 호출                          |
    |     |                                                |
    |     |-- data_agent.py                                |
    |     |     1. quick_look_data 재사용 (API 호출 없음!)   |
    |     |     2. 섹터 평균 PE만 추가 수집 (FMP)             |
    |     |     3. Claude에게 "이 숫자들 해석해줘"            |
    |     |     4. 결과: 밸류에이션, 기술지표 해석, 재무 건전성 |
    |     |     => Claude 1회 호출                          |
    |     |                                                |
    |     |-- macro_agent.py                               |
    |     |     1. FRED API에서 금리/CPI/실업률 수집          |
    |     |     2. Claude에게 "경제 상황 분석해줘"            |
    |     |     3. 결과: 금리 전망, 물가, 섹터 트렌드          |
    |     |     => Claude 1회 호출                          |
    |     |                                                |
    |     +------------------------------------------------+
    |     3개가 동시에 실행됨 (asyncio.gather)
    |     순차: 30초 x 3 = 90초 vs 병렬: 30초 = 3배 빠름
    |
    |-- [2단계: Graceful Degradation 판정] ----------------+
    |     |                                                |
    |     |  성공 3/3 => 정상 진행                           |
    |     |  성공 2/3 => 진행, 신뢰도 1단계 하향              |
    |     |  성공 1/3 => 진행, 신뢰도 low 고정               |
    |     |  성공 0/3 => 분석 중단, 에러 반환                 |
    |     +------------------------------------------------+
    |
    |-- [3단계: 교차검증]
    |     |
    |     |-- cross_validation.py
    |     |     1. 3개 Agent 결과를 한꺼번에 받음
    |     |     2. Claude에게 "모순 있는지 찾아줘"
    |     |     3. 결과: 충돌 항목, 합의 항목, 신뢰도 조정
    |     |     => Claude 1회 호출
    |
    |-- [4단계: 최종 판단]
    |     |
    |     |-- analyst_agent.py
    |     |     1. 모든 결과 + 교차검증 종합
    |     |     2. Claude에게 "최종 판단 내려줘"
    |     |     3. 결과: BUY/HOLD/SELL, Bull/Bear case
    |     |     => Claude 1회 호출
    |
    v
  analysis_state (최종 결과 dict)

[모든 Claude 호출 경로]

  news_agent.py ------+
  data_agent.py -------+-- claude_client.py ---> Claude API
  macro_agent.py ------+       |
  cross_validation.py -+       |-- 호출마다 usage_tracker.increment()
  analyst_agent.py ----+       |-- JSON 파싱 실패 시 raw_output 저장
                               |-- 하루 100회 하드 리밋 체크

[결과물: analysis_state (dict)]

  analysis_state = {
      "ticker": "NVDA",
      "quick_look_data": {...},

      "agent_results": {
          "news":  {"sentiment": "positive", "earnings": "beat", ...},
          "data":  {"valuation": "undervalued", "rsi": "overbought", ...},
          "macro": {"rate_outlook": "hold", "sector_trend": "growth", ...},
      },

      "agent_status": {
          "news": "success",
          "data": "success",
          "macro": "success",
      },

      "cross_validation": {
          "conflicts": ["기술지표 과매수 vs 실적 호조"],
          "agreements": ["성장 모멘텀 3개 Agent 모두 동의"],
      },

      "analyst": {
          "verdict": "BUY",
          "confidence": "high",
          "bull_case": "AI 수요 폭발 + 실적 서프라이즈",
          "bear_case": "밸류에이션 부담 + 금리 리스크",
          "summary": "성장 모멘텀이 밸류에이션 부담을 상회",
          "disclaimer": "본 분석은 AI가 생성한 참고 자료이며...",
      },

      "errors": [],
  }

4. STEP 3 — 화면 표시 (Phase 6~8에서 구현)
================================================================================

[app.py가 analysis_state를 받아서 화면에 배치]

  +-----------------------------------------------------------+
  |  종합 판단: BUY              신뢰도: HIGH                  |
  |------------------------------------------------------------|
  |                                                            |
  |  [뉴스 탭]        <- agent_results["news"]                 |
  |  "감성: 긍정적, 실적 서프라이즈..."                         |
  |                                                            |
  |  [재무 탭]        <- agent_results["data"]                 |
  |  "PER 35 -> 섹터 평균 대비 저평가..."                       |
  |                                                            |
  |  [경제 탭]        <- agent_results["macro"]                |
  |  "금리 동결 예상, 반도체 섹터 성장세..."                     |
  |                                                            |
  |  [교차검증]       <- cross_validation                      |
  |  "주의: 기술지표 과매수 vs 실적 호조"                       |
  |                                                            |
  |  Bull: AI 수요 폭발          <- analyst["bull_case"]       |
  |  Bear: 밸류에이션 부담        <- analyst["bear_case"]       |
  |                                                            |
  |  "본 분석은 투자 자문이 아닙니다"                           |
  +-----------------------------------------------------------+


================================================================================
5. 파일 간 역할 분담 정리
================================================================================

[파일]                    [역할]                         [비유]
--------------------------------------------------------------------------
app.py                    입력 받기 + 결과 보여주기       접수 창구
orchestrator.py           분석 흐름 제어 + 실패 처리      팀장
news_agent.py             뉴스 감성 분석                  뉴스 담당자
data_agent.py             재무/기술지표 해석              재무 담당자
macro_agent.py            거시경제 분석                   경제 담당자
cross_validation.py       모순 탐지                      감사관
analyst_agent.py          최종 BUY/HOLD/SELL 판단         수석 애널리스트
claude_client.py          Claude API 호출 공통 함수       전화기
api_client.py             외부 데이터 API 통합 + 폴백     데이터 수집반
usage_tracker.py          일일 AI 사용량 추적             계량기

[핵심 설계 원칙]
- app.py는 분석 로직을 전혀 모름 (UI만 담당)
- orchestrator.py는 화면을 전혀 모름 (로직만 담당)
- 각 Agent는 자기 분야만 담당 (다른 Agent 존재를 모름)
- 모든 Claude 호출은 claude_client.py 한 곳을 통과
- quick_look_data는 한 번만 수집되고 모든 곳에서 재사용


================================================================================
6. 비용 요약
================================================================================

[1회 Deep Analysis]
  Claude 호출: 5회
  예상 토큰: Input ~11,700 + Output ~3,500
  예상 비용: ~$0.09 (약 130원)

[일일 사용 시나리오]
  가볍게 (5회/일)  => 월 $9.9   (약 14,000원)
  보통   (10회/일) => 월 $19.8  (약 28,000원)
  적극적 (20회/일) => 월 $39.6  (약 56,000원)
  하드 리밋 (100회/일) => 월 $198 (약 280,000원)

[비용 절약 장치]
  - 일일 100회 하드 리밋
  - Data Agent는 quick_look_data 재사용 (API 재호출 없음)
  - 캐싱 (같은 종목 1시간 이내 재분석 시 캐시 사용)
  - Quick Look은 AI 미사용 (무료)
  - 80회 도달 시 경고 표시
