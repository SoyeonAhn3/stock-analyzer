# Portfolio 기능 — Stitch 화면 설계 프롬프트

## 이 앱은 뭐하는 앱인가

미국 주식 분석 대시보드다. 사용자가 종목을 검색하면 시세, 재무지표, 기술지표, AI 분석을 보여준다. 여기에 "포트폴리오" 기능을 추가한다. 사용자가 자기가 산 주식 종목, 수량, 매수가를 입력하면 현재가를 자동으로 가져와서 수익률을 계산해주고, AI가 포트폴리오 전체를 분석해서 리스크와 리밸런싱 제안을 해준다.

---

## 화면 1: 포트폴리오 메인 페이지

이 페이지에 들어가면 위에서 아래로 이런 정보가 보인다.

### 상단 요약 영역

총 3개 숫자가 보인다.

- **총 평가금액**: 내가 가진 모든 주식의 현재 가치 합계. 예: $45,230.50
- **총 투자금액**: 내가 실제로 투자한 돈 합계. 예: $38,500.00  
- **총 수익**: 평가금액 - 투자금액. 예: +$6,730.50 (+17.5%). 수익이면 초록색, 손실이면 빨간색.

이 숫자들은 보유 종목의 현재가가 바뀌면 자동으로 갱신된다.

### 차트 영역

두 개의 차트가 나란히 보인다.

**비중 도넛 차트**: 내 포트폴리오에서 각 종목이 몇 %를 차지하는지 도넛 형태로 보여준다. 예를 들어 AAPL 35%, NVDA 28%, MSFT 22%, TSLA 15%. 도넛 가운데에는 총 종목 수가 표시된다.

**수익률 가로 바 차트**: 종목별 수익률을 가로 막대로 비교한다. 수익인 종목은 초록 바, 손실인 종목은 빨간 바. 수익률이 높은 순서로 정렬된다.

### 보유 종목 목록

내가 등록한 종목들이 카드 형태로 쭉 나열된다. 각 카드에는 이런 정보가 있다.

- 종목 코드와 회사명 (예: AAPL / Apple Inc.)
- 보유 수량과 매수가 (예: 10주 × $150.00)
- 현재가 (실시간 자동 조회)
- 평가금액 (현재가 × 수량)
- 수익/손실 금액과 퍼센트 (예: +$425.00, +28.3%)

카드 오른쪽에 [⋯] 버튼이 있다. 누르면 메뉴가 나온다:
- **Quick Look**: 해당 종목의 상세 분석 페이지로 이동
- **Edit**: 수량이나 매수가를 수정하는 모달이 열림
- **Delete**: 포트폴리오에서 이 종목을 삭제. "정말 삭제하시겠습니까?" 확인 후 삭제.

카드 자체를 클릭해도 해당 종목의 Quick Look 페이지로 이동한다.

목록 상단에 정렬 기능이 있다. 수익률순, 평가금액순, 종목명순으로 정렬할 수 있다.

### 포트폴리오가 비어 있을 때

아직 종목을 하나도 추가하지 않았으면 차트나 목록 대신 "아직 추가된 종목이 없습니다. 첫 종목을 추가해보세요." 안내 문구와 "+ Add Stock" 버튼만 보인다.

## 화면 2: 종목 추가/수정 모달

메인 페이지 오른쪽 상단의 "+ Add" 버튼을 누르면 모달이 뜬다.

입력 필드 5개가 있다.

1. **종목 코드** (필수): "AAPL" 같은 티커를 입력한다. 입력하면 드롭다운으로 종목 후보가 뜨고, 선택하면 아래에 "Apple Inc. — Technology" 같은 회사명이 표시된다.
2. **보유 수량** (필수): 숫자만 입력. 예: 10
3. **평균 매수가** (필수): 달러 기준. 예: 150.00
4. **매수일** (선택): 날짜 선택. 안 넣어도 된다.
5. **메모** (선택): 자유 텍스트. 예: "실적 발표 전 매수". 최대 100자.

하단에 "취소"와 "저장" 버튼이 있다. 필수 3개(종목, 수량, 매수가)를 다 채워야 저장 버튼이 활성화된다.

수정 모드에서는 종목 코드 필드가 잠겨있고(변경 불가), 나머지만 수정할 수 있다. 저장 버튼 텍스트가 "Update"로 바뀐다.

## 화면 3: AI 포트폴리오 분석

메인 페이지 하단에 위치한다. 별도 페이지가 아니라 같은 페이지 아래쪽에 있다.

### 분석 전

"Analyze My Portfolio" 버튼이 있다. 종목이 2개 이상이어야 활성화된다. 1개 이하면 "2개 이상의 종목이 필요합니다" 안내가 뜬다.

### 분석 중

버튼이 사라지고 "AI가 포트폴리오를 분석하고 있습니다..." 로딩 표시가 나온다. 약 1~2분 걸린다.

### 분석 완료

AI가 3가지를 알려준다.

**리스크 평가**: 포트폴리오가 얼마나 위험한지. 예: "상위 2종목(AAPL+NVDA)이 전체의 63%를 차지. Technology 섹터 비중 78%로 섹터 리스크 존재." 집중도를 High/Medium/Low로 표시.

**섹터 분포**: 어떤 섹터에 얼마나 투자했는지 가로 바로 보여준다. 예: Technology 78%, Automotive 15%, Software 7%.

**제안 사항**: AI가 리밸런싱 제안을 불릿 리스트로 알려준다. 예:
- 방어주(Consumer Staples, Utilities) 편입 검토
- TSLA 손절 or 비중 축소 고려 (-8% 손실 중)
- 배당주 추가로 현금흐름 안정화 가능

맨 아래에 "AI가 생성한 참고 자료이며 투자 조언이 아닙니다" 면책 조항이 있다.

분석 결과 옆에 "마지막 분석: 2시간 전" 시각과 "재분석" 버튼이 있다.

## 화면 4: Settings — 동기화 설정

기존 Settings 페이지에 "PORTFOLIO SYNC" 섹션이 추가된다.

### 동기화란?

포트폴리오 데이터는 기본적으로 내 브라우저에만 저장된다. 그래서 집 PC에서 입력한 포트폴리오를 회사 PC에서 볼 수 없다. 동기화를 설정하면 서버를 통해 다른 기기에서도 같은 포트폴리오를 볼 수 있다. 회원가입이나 로그인은 필요 없고, 12자리 동기화 코드와 4자리 PIN만 사용한다.

### 동기화를 아직 안 했을 때

두 개 버튼이 보인다.
- **"Create Sync Code"**: 새 동기화 코드를 만든다 (주로 처음 쓰는 기기에서).
- **"Enter Existing Code"**: 다른 기기에서 이미 만든 코드를 입력한다.

### "Create Sync Code" 눌렀을 때

모달이 뜬다.

1. 4자리 PIN을 입력한다 (숫자만).
2. PIN을 한 번 더 입력해서 확인한다. 두 값이 다르면 "PIN이 일치하지 않습니다" 에러.
3. "Generate Code" 버튼을 누르면 서버가 랜덤 코드를 생성한다.
4. 화면에 "ABCD-1234-EFGH" 같은 12자리 코드가 크게 표시된다.
5. "Copy" 버튼으로 복사할 수 있다.
6. "이 코드를 메모해 두세요. 다른 기기에서 이 코드와 PIN을 입력하면 포트폴리오가 동기화됩니다." 안내 문구.
7. "Done" 버튼으로 모달을 닫는다.

### "Enter Existing Code" 눌렀을 때 (다른 기기에서)

모달이 뜬다.

1. 동기화 코드 12자리를 입력한다.
2. PIN 4자리를 입력한다.
3. "Sync Now" 버튼을 누른다.
4. 코드가 틀리거나 PIN이 틀리면 "코드 또는 PIN이 올바르지 않습니다" 에러. 3회 연속 실패하면 30초 대기.
5. 성공하면 "Sync Complete! 4개 종목을 가져왔습니다." 완료 화면.
6. "Go to Portfolio" 버튼으로 포트폴리오 페이지로 이동.

### 동기화 설정 완료 후

Settings에서 이런 정보가 보인다.
- 내 동기화 코드 (복사 가능)
- 마지막 동기화 시각 (예: "3분 전")
- "Sync Now" 버튼 (수동 동기화)
- 연결 상태: "Connected"
- "Disconnect" 버튼: 동기화를 해제한다. 누르면 "동기화를 해제하면 이 기기의 포트폴리오만 유지됩니다. 서버 데이터는 삭제됩니다." 확인 후 해제.

### 동기화 동작 원리

- 포트폴리오를 추가/수정/삭제하면 브라우저 저장과 동시에 서버에도 자동 저장된다.
- 다른 기기에서 페이지를 열면 서버에서 최신 데이터를 가져온다.
- 두 기기에서 동시에 수정하면 나중에 저장한 쪽이 이긴다 (last-write-wins).
- 90일간 접속하지 않으면 서버 데이터가 자동 삭제된다.

## 모바일에서 달라지는 점

- 요약 카드 3개가 세로로 쌓인다 (PC에서는 가로).
- 도넛 차트와 수익률 차트가 세로로 쌓인다 (PC에서는 가로).
- 종목 카드가 압축 레이아웃으로 바뀐다: 1줄에 종목코드+수익률, 2줄에 수량+현재가, 3줄에 수익금액.
- 동기화 설정 버튼 2개가 세로로 쌓인다.

## 데이터 정리

### 사용자가 입력하는 데이터
- 종목 코드 (AAPL, NVDA 등)
- 보유 수량 (정수)
- 평균 매수가 (달러, 소수점 2자리)
- 매수일 (선택)
- 메모 (선택, 100자 이내)

### 자동으로 계산되는 데이터
- 현재가: 서버에서 60초마다 자동 조회
- 평가금액: 현재가 × 수량
- 투자금액: 매수가 × 수량
- 수익금액: 평가금액 - 투자금액
- 수익률: (수익금액 / 투자금액) × 100
- 비중: 해당 종목 평가금액 / 전체 평가금액 × 100

### 저장 위치
- 기본: 브라우저 LocalStorage (서버 전송 안 함 → 다른 사람 못 봄)
- 동기화 설정 시: 브라우저 + 서버 동시 저장 (동기화 코드+PIN 없이는 접근 불가)
