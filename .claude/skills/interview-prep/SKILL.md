# interview-prep Skill

프로젝트 전체를 분석하고, JD/지원서와 크로스 매칭하여 면접 예상 질문 및 답변 가이드를 생성한다.
출력 파일 구조는 `references/template.md`를 참고한다.

---

## STEP 0 — 레퍼런스 로드

스킬 실행 전 반드시 먼저 실행:

```
Read(".claude/skills/interview-prep/references/template.md")
```

→ 파일 경로 규칙, 내용 구조, 채팅 출력 포맷 확인

---

## STEP 1 — 입력 수집

사용자에게 아래 2가지를 확인:

### 1-A. 대상 프로젝트
- 사용자가 프로젝트 경로를 지정하면 해당 경로 사용
- 미지정 시 현재 작업 디렉토리를 대상 프로젝트로 사용

### 1-B. JD/지원서
아래 중 하나로 입력받음:
- **파일 경로 지정**: 이미지(png, jpg) 또는 PDF 파일 → Read 도구로 읽기
- **채팅 직접 입력**: 사용자가 텍스트로 붙여넣기

둘 다 없으면 한 번만 질문:
```
"면접 준비를 위해 아래 정보를 알려주세요:
1. 대상 프로젝트 경로 (미입력 시 현재 프로젝트)
2. JD 또는 지원서 (파일 경로 또는 직접 붙여넣기)"
```

---

## STEP 2 — 프로젝트 분석

대상 프로젝트를 아래 순서로 분석:

1. **구조 파악**: Glob으로 디렉토리 구조, 주요 파일 목록
2. **기술 스택**: package.json, requirements.txt, import문 등에서 추출
3. **핵심 로직**: 주요 소스 코드 Read
4. **문서**: README.md, Phase 문서, 매뉴얼 등
5. **테스트 결과**: test-scenarios/, logs/ 폴더 확인
6. **Git 히스토리**: `git log --oneline -20`으로 개발 흐름 파악

---

## STEP 3 — JD/지원서 분석

입력받은 JD/지원서에서 아래 항목 추출:

| 추출 대상 | 설명 |
|-----------|------|
| 필수 요구사항 | 지원 자격에 명시된 기술/경험 |
| 우대사항 | 우대하는 추가 역량/경험 |
| 직무 내용 | 실제 수행할 업무 설명 |
| 기술 스택 | JD에 명시된 기술 목록 |
| 핵심 키워드 | 반복적으로 강조되는 역량/가치 |

---

## STEP 4 — 질문 및 답변 생성

### 질문 카테고리 (5개)

| 카테고리 | 비율 | 의미 |
|----------|------|------|
| 기술 심화 | 30% | 기술 스택, 아키텍처, 설계 결정 |
| 문제 해결 | 25% | 개발 중 어려움, 버그 해결, 트레이드오프 |
| 역할/기여도 | 20% | 본인 역할, 기여 범위, 협업 |
| JD 연결 | 15% | JD 요구사항과 프로젝트 경험 연결 |
| 확장/개선 | 10% | 한계점, 개선 방향, 향후 계획 |

### 질문 난이도

| 난이도 | 비율 | 설명 |
|--------|------|------|
| Basic | 30% | 프로젝트 개요, 기본 기술 설명 |
| Intermediate | 40% | 설계 결정 이유, 비교 분석 |
| Advanced | 30% | 깊은 기술 이해, 트레이드오프, 대안 제시 |

### 답변 작성 규칙

**STAR 구조 기반:**
- **Situation**: 프로젝트 배경/맥락
- **Task**: 맡은 과제/문제
- **Action**: 구체적으로 한 행동 (핵심)
- **Result**: 결과/성과/배운 점

**필수 포함:** 프로젝트 코드/구조의 구체적 근거, 기술 선택 이유
**피할 것:** 불확실한 표현, 확인 안 된 내용 추측

### 꼬리 질문
각 질문마다 2개씩 꼬리 질문 대비 포함

---

## STEP 5 — 파일 저장

`references/template.md`의 파일 경로 규칙과 내용 구조에 맞게 저장.

1. `interview/` 폴더 없으면 생성: `mkdir -p interview`
2. `template.md`의 파일 내용 구조로 마크다운 작성
3. Write 도구로 저장

---

## STEP 6 — 채팅 출력

`references/template.md`의 채팅 출력 포맷에 맞게 저장 결과 + 전체 내용 출력.

---

## 실패 처리

| 실패 유형 | 처리 방법 |
|----------|----------|
| JD/지원서 미제공 | STEP 1 질문 1회 실행 |
| 파일 읽기 실패 (이미지/PDF) | 파일 경로 재확인 요청 |
| 프로젝트 경로 없음 | 현재 디렉토리 사용 안내 |
| 폴더 생성 실패 | Bash 재시도 → 실패 시 채팅에만 출력 |
| 파일 저장 실패 | 채팅에만 출력 후 저장 실패 안내 |

---

## 주의사항

- 프로젝트에서 확인되지 않는 내용은 추측하지 않음
- 답변은 프로젝트 코드/문서에서 실제 발견한 근거 기반으로만 작성
- JD가 없이 프로젝트만 제공된 경우, JD 연결 카테고리는 생략하고 나머지 4개 카테고리로 생성
